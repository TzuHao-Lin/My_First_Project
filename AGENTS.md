# Codex Project Rules

## General Behavior

- Before editing code, read the relevant files first.
- Explain the intended change before modifying files.
- Keep changes focused on the user's request.
- Do not refactor unrelated code.
- Do not delete or overwrite user changes.
- Ask for clarification before risky or ambiguous changes.

## Feedback Editing

When handling review comments, grading feedback, or scattered feedback:

1. Read the original code or document first.
2. Read all related review comments or feedback.
3. Understand the real issue behind each comment.
4. Directly improve the code or document instead of only adding explanation.
5. Keep the result maintainable, readable, and consistent with the existing style.

## Git

- Do not commit unless the user explicitly asks.
- Do not push unless the user explicitly asks.
- Do not run destructive git commands unless explicitly requested.
